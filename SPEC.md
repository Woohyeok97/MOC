# MOC — MVP 스펙

> Rebrickable 스타일의 레고 도안(설명서) 마켓플레이스.  
> 창작자가 도안을 올리고, 구매자가 보고 구매해서 PDF로 다운로드하는 서비스.  
> **MVP는 무료 도안만 지원**, 결제·검색·필터 등은 이후 단계에서 추가.

---

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **서비스명** | MOC |
| **참고 서비스** | [Rebrickable](https://rebrickable.com/) |
| **한 줄 소개** | 레고 창작자가 도안을 올리고, 누구나 무료로 구매해 PDF를 다운로드할 수 있는 마켓플레이스 |
| **핵심 사용자** | 창작자 (도안 올리는 사람), 구매자 (도안 구매하는 사람) |
| **MVP 범위 요약** | 구글 로그인, 무료 도안 등록·조회·구매·다운로드 |

## 2. 요구사항

### 2.1 사용자 역할 & 권한

MVP에서는 별도의 "창작자/구매자" 계정 구분 없이, **로그인한 모든 유저가 도안 등록도 하고 구매도 가능**.

| 역할 | 할 수 있는 것 | 할 수 없는 것 |
|------|--------------|--------------|
| **비로그인** | 도안 목록 조회, 상세 페이지 조회 | 구매, 다운로드, 도안 등록 |
| **로그인 유저** | 도안 목록·상세 조회, 구매, 다운로드, 도안 등록·수정·삭제 (본인 것만) | 타인의 도안 수정·삭제 |

### 2.2 기능 요구사항

**인증**
- 구글 소셜 로그인 / 로그아웃
- 로그인 상태에 따른 UI 분기 (구매 버튼, 등록 버튼 등)
- 비로그인 상태로 구매 클릭 시 → 로그인 화면으로 이동

**도안 등록·수정·삭제**
- 로그인 유저만 도안 등록 가능
- 등록 시 입력 항목:

  | 필드 | 필수 여부 | 비고 |
  |------|-----------|------|
  | 제목 | ✅ | |
  | 설명 | ✅ | |
  | 카테고리 | ✅ | `architecture` / `castles` / `vehicles` / `robots` / `others` |
  | 이미지 | ✅ | 썸네일 겸 미리보기용, 여러 장 가능 (첫 번째가 대표 썸네일) |
  | PDF 파일 | ✅ | 도안 파일 (다운로드용), 여러 파일 가능 |
  | 가격 | — | MVP에서는 0원(무료) 고정, 별도 입력 필드 불필요 |

- 수정·삭제: **본인이 올린 도안만** 가능

**도안 목록·상세 조회**
- 목록 페이지: 등록된 전체 도안을 최신순으로 표시 (썸네일, 제목, 카테고리)
- 상세 페이지: 도안 전체 정보 + 구매/다운로드 버튼 (로그인 여부에 따라 분기)
- 검색·필터·정렬은 MVP 미포함

**구매 → 다운로드 흐름**
1. 로그인 유저가 상세 페이지에서 **「구매」** 클릭
2. DB에 "이 유저가 이 도안을 구매함" 기록 (구매 이력 저장)
3. **「구매」 버튼 → 「다운로드」 버튼으로 전환**
4. 이후 상세 페이지 재방문 시에도 「다운로드」 버튼 표시 유지
5. **「다운로드」** 클릭 시 PDF 파일 다운로드

**내가 구매한 도안 / 내가 올린 도안**
- 각각 목록 확인 및 PDF 다운로드 가능

### 2.3 페이지 & 라우팅

| URL | 페이지 | 접근 권한 |
|-----|--------|-----------|
| `/` | 홈 (도안 목록) | 누구나 |
| `/designs/[id]` | 도안 상세 | 누구나 (구매·다운로드는 로그인만) |
| `/designs/new` | 도안 등록 폼 | 로그인 필요 |
| `/designs/[id]/edit` | 도안 수정 폼 | 본인만 |
| `/my/purchases` | 내가 구매한 도안 목록 | 로그인 필요 |
| `/my/designs` | 내가 올린 도안 목록 | 로그인 필요 |

## 3. 아키텍처 결정

### 3.1 기술 스택

| 구분 | 선택 | 결정 이유 |
|------|------|-----------|
| **언어** | TypeScript | 타입 안전성, 자동완성, 런타임 오류 사전 방지 |
| **패키지 관리** | Yarn | 빠른 설치 속도, 안정적인 lockfile 관리 |
| **프레임워크** | Next.js (App Router) | 서버 컴포넌트 + API Routes로 풀스택 단일 레포 |
| **DB** | Supabase (PostgreSQL) | 관리형 DB, 인증·스토리지와 통합 운영 가능 |
| **ORM** | Prisma | 타입 안전한 DB 접근, 스키마 관리 및 마이그레이션 |
| **인증** | Supabase Auth | DB·스토리지와 동일 플랫폼, 권한 제어 연동 단순 |
| **파일 스토리지** | Supabase Storage | 인증 유저 기반 접근 권한 제어 (Signed URL) 기본 지원 |
| **스타일** | Tailwind CSS + shadcn/ui | 빠른 UI 구성, 컴포넌트 재사용 |
| **폼 관리** | React Hook Form + Zod | 폼 상태 관리 및 유효성 검사 타입 안전하게 처리 |
| **서버 상태** | TanStack Query | 데이터 패칭·캐싱·동기화 |
| **전역 상태** | Zustand | 로그인 유저 정보 등 클라이언트 상태 관리 |
| **배포** | 미정 | |

### 3.2 주요 아키텍처 결정

- **인증 흐름**: 구글 OAuth → 토큰 발급 → Supabase users 테이블과 연동
- **파일 업로드**: 클라이언트에서 직접 Supabase Storage에 업로드 후 URL을 DB에 저장
- **다운로드 권한 제어**: `purchases` 테이블 조회로 구매 여부 확인 후 서명된 URL(Signed URL) 또는 직접 URL 반환
- **MVP 가격 처리**: `price` 컬럼은 DB에 존재하지만 MVP에서는 항상 0으로 저장, UI에는 노출하지 않음

## 4. 데이터 모델

> Prisma ORM을 통해 Supabase(PostgreSQL)에 접근.

### users
```
id    text  (PK)  -- Supabase Auth UUID (auth.users.id와 동일)
name  text        -- 표시 이름 (구글 프로필에서 가져옴, 타인 도안의 작성자명 표시용)
```
- 구글 소셜 로그인 시 생성·연동
- 이메일·프로필 이미지는 Supabase Auth(`auth.users`)가 관리하므로 별도 저장하지 않음

### designs (도안)
```
id            uuid      (PK)              -- 도안 고유 식별자
title         text                        -- 도안 제목
description   text                        -- 도안 설명
category      DesignCategory              -- 카테고리 enum
price         integer   (MVP: 0 고정)     -- 가격 (MVP는 0 고정, 이후 유료 확장 대비)
instructions  text[]                      -- PDF 파일 URL 배열 (여러 파일 가능)
images        text[]                      -- 이미지 URL 배열 (여러 장, 첫 번째가 대표 썸네일)
author_id     text      (→ users.id)      -- 도안을 등록한 유저 (작성자)
created_at    timestamp                   -- 도안 등록 시각
updated_at    timestamp                   -- 도안 최근 수정 시각
```

### purchases (구매 이력)
```
id            uuid     (PK)              -- 구매 이력 고유 식별자
user_id       text     (→ users.id)      -- 구매한 유저
design_id     uuid     (→ designs.id)    -- 구매한 도안
amount        integer  (MVP: 0 고정)     -- 구매 시점 금액 스냅샷 (가격 변경 대비)
purchased_at  timestamp                  -- 구매 시각
```
- "이 유저가 이 도안을 구매했는가" 판단 및 다운로드 권한 확인용
- `(user_id, design_id)` 조합은 유니크 제약 (중복 구매 방지)

## 5. 테스트 전략

### MVP 테스트 범위

MVP 단계에서는 **핵심 흐름을 수동으로 직접 검증**하는 방식으로 진행.  
자동화 테스트(E2E, 단위 테스트)는 이후 단계에서 추가.

### 각 기능별 검증 항목

| 기능 | 검증 항목 |
|------|-----------|
| **구글 로그인** | 로그인 후 유저 정보가 DB에 저장되는지, 로그아웃 후 상태가 초기화되는지 |
| **도안 등록** | 이미지·PDF 업로드 후 스토리지 저장 및 URL이 DB에 기록되는지 |
| **도안 조회** | 비로그인·로그인 상태 각각에서 목록·상세 페이지가 올바르게 표시되는지 |
| **구매 흐름** | 구매 클릭 후 `purchases`에 기록되고 버튼이 다운로드로 바뀌는지 |
| **다운로드** | 구매한 도안만 다운로드 가능한지, 구매하지 않은 도안은 다운로드 불가한지 |
| **권한 분기** | 비로그인 유저가 구매 클릭 시 로그인 페이지로 이동하는지 |
| **도안 수정·삭제** | 본인 도안만 수정·삭제 가능하고, 타인 도안 접근 시 차단되는지 |

### 이후 단계 자동화 목표
- 핵심 흐름에 대한 E2E 테스트 (Playwright 등)
- API Route 단위 테스트

## 6. MVP 범위

### In Scope (MVP에서 하는 것)
- 구글 소셜 로그인/로그아웃
- 도안 등록·수정·삭제 (이미지 + PDF 업로드)
- 도안 목록 조회 (최신순)
- 도안 상세 조회
- 구매 → 다운로드 흐름 (무료)
- 내가 구매한 도안 목록
- 내가 올린 도안 목록

### Out of Scope (MVP에서 하지 않는 것)
- 유료 결제 (토스페이먼츠, Stripe 등)
- 검색, 카테고리 필터, 정렬 옵션
- 이메일 알림 / 이메일로 다운로드 링크 발송
- 구글 외 소셜 로그인 (카카오 등)
- 모바일 앱 (웹 반응형만)
- 자동화 테스트 (E2E, 단위 테스트)

### 이후 단계 로드맵
1. 유료 도안 등록 + 결제 연동 (토스페이먼츠/Stripe)
2. 카테고리 필터, 검색, 정렬
3. 추가 소셜 로그인
4. 이메일 알림/다운로드 링크
5. E2E 및 단위 테스트 자동화
6. 배포 환경 확정 및 CI/CD
