> 각 Step의 상세 내용(기술 구조, 엣지 케이스, 검증 항목)은 해당 Step 구현 직전에 이 파일에 추가한다.

## Step 1. 프로젝트 초기 세팅

**범위**
- Next.js 프로젝트 생성 (TypeScript, App Router, Yarn)
- Tailwind CSS + shadcn/ui 설치 및 설정
- ESLint / Prettier 설정
- 폴더 구조 잡기
- 환경변수 파일 (.env.local) 구성

---

## Step 2. Supabase + Prisma 세팅

**범위**
- Supabase 프로젝트 생성
- Prisma 설치 및 Supabase(PostgreSQL) 연결
- DB 스키마 작성 (users, designs, purchases)
- 마이그레이션 실행 및 연결 확인

---

## Step 3. Supabase Auth + 구글 로그인

**범위**
- Supabase Auth 설정 (구글 OAuth)
- `@supabase/ssr` 세션 관리 연동
- 로그인 / 로그아웃 기능 구현
- 로그인 상태에 따른 UI 분기 (헤더, 버튼 등)
- 비로그인 상태로 보호된 페이지 접근 시 로그인 페이지로 이동

---

## Step 4. 도안 목록 페이지 (`/`)

**범위**
- 전체 도안 목록 조회 (최신순)
- 썸네일, 제목, 카테고리 표시
- 반응형 그리드 레이아웃

---

## Step 5. 도안 등록 폼 (`/designs/new`)

**범위**
- 로그인 유저만 접근 가능
- 입력 항목: 제목, 설명, 카테고리, 이미지(여러 장), PDF
- React Hook Form + Zod 유효성 검사
- Supabase Storage에 이미지·PDF 업로드
- 업로드 후 URL을 DB에 저장

---

## Step 6. 도안 상세 페이지 (`/designs/[id]`)

**범위**
- 도안 전체 정보 표시 (이미지, 제목, 설명, 카테고리)
- 로그인 여부에 따른 버튼 분기
  - 비로그인: 버튼 비활성화 또는 로그인 유도
  - 로그인 + 미구매: 구매 버튼
  - 로그인 + 구매 완료: 다운로드 버튼
- 본인 도안일 경우 수정·삭제 버튼 표시

---

## Step 7. 구매 → 다운로드 흐름

**범위**
- 구매 버튼 클릭 → purchases 테이블에 기록
- 버튼 상태 전환 (구매 → 다운로드)
- 다운로드 클릭 → Supabase Storage Signed URL 반환 후 PDF 다운로드
- 페이지 재방문 시에도 구매 상태 유지

---

## Step 8. 도안 수정·삭제 (`/designs/[id]/edit`)

**범위**
- 본인이 올린 도안만 접근 가능 (타인 접근 시 차단)
- 기존 데이터 불러와서 폼에 표시
- 수정: 항목 변경 + 파일 교체 가능
- 삭제: 도안 및 Storage 파일 삭제

---

## Step 9. 내 도안 목록 페이지 (`/my/purchases`, `/my/designs`)

**범위**
- `/my/purchases`: 내가 구매한 도안 목록 + 다운로드
- `/my/designs`: 내가 올린 도안 목록 + 수정·삭제 링크
- 로그인 유저만 접근 가능

---
